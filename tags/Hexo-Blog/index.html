由于笔记处于未开发完成状态，不希望对外。但由于Github不支持公共仓库为分支单独设置可见性权限

因此，无法使用特定分支隐藏不希望对外的内容。因此，我们需要借助Github Actions来实现自动部署

笔记+博客的源码使用私有仓库进行管理，每当私有仓库发生提交后，利用Github Actions自动构建并推送至Github Page对应的公开仓库

首先定义下仓库结构，目前使用两个仓库：

  • blog-source：保存博客的源文件
  • lonelywaiting.github.io：最终的发布仓库

首先生成SSH密钥：若已生成过，则可忽略。在.ssh目录下生成id_ed25519与id_ed25519.pub

然后配置blog-source仓库：

  • 打开Settings/Actions/General，修改Actions Permissions为Allow all actions and reusable workflows。便于后续复用Action，否则会出现以下报错

  • 打开Settings/Secrets and variables/Actions，点击New repository secret，添加SSH私钥，命名为SSH_DEPLOY_KEY

接下来配置发布仓库：添加Deploy Keys，有两种方式可以选择

  • 一：仓库Settings/Security/Deploy keys/ Add deploy key，添加SSH公钥
  • 二：用户Settings/SSH And GPG keys/New SSH key，添加SSH公钥
